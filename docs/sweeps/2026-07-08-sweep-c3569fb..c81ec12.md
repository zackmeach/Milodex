### SWP-20260708-01 — heal write escapes the #320 fail-safe (MED, reliability)

`_heal_stale_adjustment`'s try/except spans only the refetch call (lines 435–443); `_bars_to_df(bars)` (447) and `self._cache.write(...)` (448) are outside it, and the call site in the `get_bars` full-cache-hit path (141–143) has no enclosing guard. A parquet-write IOError (the documented Windows file-lock hazard) propagates out of a warm read whose own contract (lines 137–139) promises "never a new failure mode". The pre-guard path did zero writes, so this is a genuinely new failure mode on the live daily poll loop. Trigger is narrow (real corp-action divergence AND concurrent write contention) but the contract violation is direct. Fix: widen the try to cover parse + write, WARN + serve cache on failure.

### SWP-20260708-02 — CLAUDE.md session-guard gotcha lies post-#315 (MED, contract-drift)

CLAUDE.md:84 states the guard as `latest_bar.timestamp.date() >= _now().date()`, "both UTC" — present tense, as the live contract. #315 (`ac3bcd2`) rewrote `runner.py:666-667` so the *now* side is ET (`self._now().astimezone(ET_TZ).date()`); the runner docstring explicitly says the now side MUST be ET and documents the 20:00–24:00 ET bug the UTC form caused. #315 never touched the bullet. An agent grounding on the gotcha misdiagnoses evening-launch behavior — or "reconciles" code back to UTC, reinstating the fixed bug. Fix: correct the snippet + replace "both UTC" with "bar-date UTC vs now-date ET". (One additional stale copy in `docs/superpowers/plans/2026-05-28-daily-runner-repair.md` — dated historical plan, leave as-is per convention.)

### SWP-20260708-03 — FRONT renders a fabricated quiet day under its own error banner (MED, surface)

FRONT wires the SectionStatus banner (FrontSurface.qml:158-163) but — unlike DESK, which gates every section (`DeskSurface.qml:695/738/857/947` `visible: …dataStatus !== "error"`) — gates no content on `dataStatus`/`hasSummaryData`. Property defaults render a plausible calm state on error/first-load: `$0.00`/"Flat"/"No realized movement", all-zero stage tally, "No strategy needs attention", "00 on the bench". The error state is reachable (any builder raise → `polling_lifecycle.py:185-190` → `dataStatus="error"`). Operator sees the honest banner AND a fabricated quiet day simultaneously — the calm landing surface asserting "nothing needs you" is exactly the wrong surface to fabricate. Fix: port DESK's hide-on-error contract to FRONT's content blocks.

### SWP-20260708-04 — `ruff` is the one unpinned dev dep; local vs CI format-gate divergence (LOW-MED, toolchain)
