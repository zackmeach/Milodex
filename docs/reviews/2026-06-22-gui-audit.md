# Milodex GUI Walkthrough Audit — 2026-06-22

Hands-on walkthrough of the running GUI (`Milodex gui`, not the packaged prototype),
driven via computer-use. Every section, drawer, per-row action menu, evidence panel,
and filter was exercised. App ran with **no console errors** for the whole session.

This is a point-in-time observation log, not a plan. Stage counts and metrics below
were live as of the session (paper mode, market closed, aggressive risk profile).

---

## Sections walked

- **FRONT** — daily briefing
- **BENCH** — 139-strategy ladder across 5 lifecycle tiers (idle / backtest / paper / micro-live / live)
- **LEDGER** — governance log with GROUP (Promotion / Lifecycle / Backtest / System) + FILTER (all / paper / clear)
- **DESK** — 7-panel live-ops dashboard
- **Risk drawer** (top-right chip) — fleet reconciliation, risk profile, time format, runner-health poll, quit
- **Bench row action menu** (Initiate Backtest / Open Evidence) + **Evidence Snapshot** drawer

---

## What's working

- **Design language.** Editorial/newspaper metaphor is distinctive, cohesive, legible across all four sections. Not generic-AI UI.
- **Posture is unmissable.** Persistent header (`RISK OFFICE · AGGRESSIVE | GUARD READY | PAPER ONLY | BROKER CONNECTED | NO REAL EXPOSURE`) + red `ELEVATED POSTURE` banner. Correct safety instinct.
- **DESK density.** Risk/mode, P/L with window tabs, per-runner liveness, the risk-gate funnel, attention list, market tape, order tape with filters — genuinely strong operational density.
- **Governance trail is real.** LEDGER Promotion filter shows `backtest → paper` PROMOTED / DEMOTED / RETURNED with operator rationale and OPEN RECORD links.
- **Cross-view consistency.** FRONT market = DESK tape; FRONT "at the gate" Tsmom 0.88 = BENCH paper Tsmom.

## Findings (by severity)

### Trust & mechanics

1. **Evidence is non-authoritative.** Evidence Snapshot states it plainly: `Authoritative: false`,
   `Freshness / Gate result: not_reconstructed_v1`, *"Real event-derived freshness and gate
   reconstruction are deferred."* The operator's primary evidence surface is a read-model
   snapshot, not reconstructed from the event store. Core gap for an evidence-driven system.
2. **Rejections have no reason.** DESK → REJECTIONS shows `tsmom · INTC submit/blocked` — no veto
   rule named. This week's funnel: 588 evals → 14 signals → **3 approved / 11 rejected → 0 filled**.
   No visibility into *which* guardrail is blocking everything.
3. **Phantom runner counted as live.** ACTIVE OPERATIONS reads "11 runners," but the inspected one
   is `SESSION STOPPED / HEARTBEAT no activity / LOCK RELEASED / AGE 63h`. Count contradicts the
   liveness fields (the `ended_at IS NULL` phantom pattern leaking into the UI).
4. **No manual halt in the GUI.** Risk drawer has profile / reconciliation / time / poll / quit —
   no emergency "stop all" / kill-switch trip. Reset presumably only appears when tripped, but
   there's no surfaced manual halt path.

### Correctness / stale copy

5. **Title collisions everywhere.** Dozens of rows render as identical bold `Intraday` / `Dia` /
   `Gld` / `Spy` / `Xl*`; only the grey strategy-id subtitle disambiguates. Hits BENCH, LEDGER,
   and the DESK attention list. Display name is taking the variant token, not `family.template.variant`.
6. **Stale references.** BENCH footer: *"Kill switch is a global affordance on the Anchor view"* —
   AnchorSurface was deleted. `MICRO LIVE / LIVE locked in Phase 5` is ambiguous (phases 1–5 closed).
   Evidence drawer's "Available Bench Actions" prose (*"only Open Evidence is wired"*) contradicts
   its own labels (*Initiate Backtest = Invocation*) **and** the Ledger's real
   `Initiate Backtest via Bench GUI` events.
7. **FRONT count doesn't reconcile.** Headline "139 strategies"; the four buckets (93+43+0+0) sum
   to **136** — the 3 IDLE strategies are dropped.
8. **IDLE semantics.** Header says "configured, not yet run," but rows show Sharpe / Max-dd /
   769 trades / "gate failing." Can't be both.
9. **Clock mismatch.** DESK panels read `as of 08:39` while system clock is `12:39 AM` (~8h offset,
   likely UTC-vs-local) — undercuts "snapshot freshness stated plainly."

### Usability

10. **Bench is mostly scaffolding.** ~93 backtest + many paper rows are benchmark/null strategies
    (`time_of_day_null`, `no_trade`, `random_matched_exposure`, `unconditional_intraday_long`).
    The ~6 real passing strategies (Atr Channel, Bbands, Donchian, Pullback Rsi2, Tsmom, Sma200
    regime) are buried, with no filter to hide nulls.
11. **Ledger default is noise.** Unfiltered + System views = a wall of identical
    `Intraday registration ADDED (config file mtime, no event-store history)`. Entries show
    **time only, no date** — ambiguous across a multi-day log.
12. **Order tape is 1 row tall.** Panel VII ("200 events") is crammed against the window bottom;
    scrolls internally but shows ~1 row. The audit trail is effectively unreadable.
13. **Dead drill-down.** FRONT `strategy detail →` is greyed and no-ops; `open in bench →` dumps
    you at the top of the 139-row list with no scroll-to/highlight of the strategy you came from.
14. **Attention misses bad actors.** `rsi2.intraday.spy` is at paper with **Sharpe −7.96** (orb/vwap
    intraday all negative), yet UNDERPERFORMING = 0 / NEEDS REVIEW = 0.

### Root-cause note

Findings #1 (non-authoritative evidence), #2 (no reject reason), and #3 (phantom liveness) share a
root cause: the GUI reads a *normalized read-model snapshot* instead of replaying the SQLite event
store, so the event-level "why" is lost. #5 (title collisions) is one fix with broad payoff — a
single display-name helper deriving from `family.template.variant` cleans BENCH, LEDGER, and DESK
at once.
